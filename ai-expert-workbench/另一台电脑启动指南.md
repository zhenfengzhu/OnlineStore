# 另一台电脑启动指南

这份文档用于把 `ai-expert-workbench` 复制到另一台 Windows 电脑后正常启动。按顺序执行即可。

## 1. 需要准备的环境

另一台电脑需要安装：

- Windows 10/11
- Node.js 22 或更高版本，建议安装 LTS/current 版本
- npm，安装 Node.js 后会自带
- PowerShell，Windows 自带

检查命令：

```powershell
node --version
npm --version
```

如果能看到版本号，说明 Node.js 和 npm 可用。

## 2. 复制项目文件

把整个项目文件夹复制到另一台电脑，例如：

```text
D:\AUnityProject\OnlineStore\ai-expert-workbench
```

建议复制这些内容：

```text
ai-expert-workbench
├─ package.json
├─ package-lock.json
├─ Start-Workbench.bat
├─ README.md
├─ 另一台电脑启动指南.md
├─ .env.example
├─ prisma
├─ scripts
└─ src
```

不需要复制这些生成目录：

```text
node_modules
.next
.launcher
logs
```

如果你想把已有产品资料、内容资产、日历和复盘记录也带过去，需要复制：

```text
prisma\dev.db
```

如果你想把当前已保存的模型配置也带过去，可以复制：

```text
.model-config.json
```

注意：`.model-config.json` 里可能有 API Key，不要发给别人。

## 3. 安装依赖

打开 PowerShell，进入项目目录：

```powershell
cd D:\AUnityProject\OnlineStore\ai-expert-workbench
```

安装依赖：

```powershell
npm install
```

这一步会下载 Next.js、React、Prisma、OpenAI SDK 等依赖。

如果下载很慢，可以切换 npm 镜像：

```powershell
npm config set registry https://registry.npmmirror.com
npm install
```

## 4. 配置环境变量

如果项目里没有 `.env`，复制一份：

```powershell
copy .env.example .env
```

最小可用 `.env`：

```env
MODEL_PROVIDER=openai
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-5.2
DATABASE_URL=file:./dev.db
```

说明：

- 如果你准备在页面里点击切换模型，可以先保留占位 API Key。
- 页面保存的模型配置会写入 `.model-config.json`，优先级高于 `.env`。
- 数据库路径建议保持 `DATABASE_URL=file:./dev.db`，对应实际文件是 `prisma\dev.db`。

## 5. 初始化数据库

首次启动前执行：

```powershell
npm run db:init
```

这个命令会创建或补齐本地 SQLite 数据库：

```text
prisma\dev.db
```

如果你已经从旧电脑复制了 `prisma\dev.db`，也要执行一次 `npm run db:init`，它会自动补齐新版本需要的字段，不会删除旧数据。

## 6. 启动方式一：点击启动

双击项目根目录：

```text
Start-Workbench.bat
```

启动器打开后点击：

```text
启动项目
```

启动器会自动执行：

```powershell
npm run db:init
npm run dev -- -p 3001
```

默认访问地址：

```text
http://localhost:3001
```

启动器里的按钮：

- `启动项目`：启动 Next.js 开发服务
- `打开浏览器`：打开 `http://localhost:3001`
- `初始化数据库`：执行 `npm run db:init`
- `停止项目`：停止启动器记录的服务进程
- `刷新状态`：检查 3001 端口是否正在运行

## 7. 启动方式二：命令启动

如果不想用启动器，可以手动启动：

```powershell
cd D:\AUnityProject\OnlineStore\ai-expert-workbench
npm run db:init
npm run dev -- -p 3001
```

然后打开：

```text
http://localhost:3001
```

如果要用生产模式：

```powershell
npm run build
npm run start -- -p 3001
```

## 8. 配置模型

启动页面后，进入左侧：

```text
模型设置
```

选择一个模型供应商，填写 API Key 和模型名，然后点击：

```text
保存并切换
```

支持：

- OpenAI
- DeepSeek
- 通义千问
- 智谱 GLM
- 豆包
- 自定义兼容接口

保存后会生成：

```text
.model-config.json
```

这个文件保存在本地，里面包含 API Key。不要提交到 git，也不要发给别人。

## 9. 推荐迁移清单

如果你想让另一台电脑和当前电脑保持一致，建议复制：

```text
package.json
package-lock.json
Start-Workbench.bat
README.md
另一台电脑启动指南.md
.env.example
.env
.model-config.json
prisma\schema.prisma
prisma\dev.db
scripts\
src\
```

然后在另一台电脑执行：

```powershell
npm install
npm run db:init
```

最后双击：

```text
Start-Workbench.bat
```

## 10. 常见问题

### 10.1 双击启动器没反应

用 PowerShell 手动运行：

```powershell
cd D:\AUnityProject\OnlineStore\ai-expert-workbench
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\launcher.ps1
```

如果提示找不到 `npm.cmd`，说明 Node.js 没装好，或没有加入 PATH。

### 10.2 端口被占用

启动器默认使用：

```text
3001
```

查看占用：

```powershell
netstat -ano | findstr :3001
```

如果被占用，可以先结束对应进程，或修改：

```text
scripts\launcher.ps1
```

把：

```powershell
$port = 3001
```

改成：

```powershell
$port = 3002
```

### 10.3 页面能打开，但生成失败

优先检查：

- 是否在“模型设置”里保存了 API Key
- 模型名是否正确
- API Key 是否有余额或权限
- 网络是否能访问对应模型接口

也可以打开：

```text
http://localhost:3001/api/model-config
```

查看当前模型是否显示：

```json
"configured": true
```

### 10.4 数据不见了

业务数据在：

```text
prisma\dev.db
```

如果没复制这个文件，另一台电脑会是一个新的空数据库。

### 10.5 依赖安装失败

先清理再安装：

```powershell
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json
npm install
```

如果网络慢：

```powershell
npm config set registry https://registry.npmmirror.com
npm install
```

### 10.6 Prisma 或 SQLite 报错

重新初始化数据库：

```powershell
npm run db:init
npm run prisma:generate
```

如果仍然失败，确认当前目录是项目根目录：

```powershell
pwd
```

应该在：

```text
...\ai-expert-workbench
```

## 11. 最短启动流程

如果环境已经装好，最短流程是：

```powershell
cd D:\AUnityProject\OnlineStore\ai-expert-workbench
npm install
npm run db:init
```

然后双击：

```text
Start-Workbench.bat
```

打开：

```text
http://localhost:3001
```
